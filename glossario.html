Glossário de Tecnologia

Não entendeu algum termo? Aqui você tira suas dúvidas.

  • A
  • API: Interface de Programação de Aplicações; um conjunto de definições e protocolos usados para permitir que softwares se comuniquem entre si.
  • Array: Estrutura de dados que armazena uma coleção de elementos que podem ser endereçados individualmente.
  • Angular: É um framework JavaScript desenvolvido pelo Google para criar aplicações web dinâmicas e interativas. Ele facilita a criação de interfaces de usuário complexas, utilizando componentes, injeção de dependências e outros recursos, além de ser altamente escalável e modular.

  • B
  • Backend: Lista com breves descrições de todas as funcionalidades desejadas para um produto específico, ainda não atribuídas a um responsável. Uma espécie de arquivo.
  • Backlog: A parte do desenvolvimento de software que lida com a lógica do servidor, bancos de dados e operações por trás das interfaces.
  • Backup: Cópia de dados de um dispositivo de armazenamento a outro para evitar perda dos dados originais.
  • Bit: É a menor unidade de medida de dados que pode ser armazenada ou transmitida no universo computacional. Um bit tem um único valor, zero ou um, com valor de verdadeiro ou de falso.
  • Branch (ou ramificação): É uma cópia independente de um código-fonte em um sistema de controle de versão, como o Git. Ela permite que você trabalhe em novas funcionalidades ou correções sem afetar o código principal, e depois pode mesclar (merge) as mudanças de volta ao branch principal (geralmente chamado de "main" ou "master").

  • C
  • C#: É uma linguagem de programação desenvolvida pela Microsoft, usada principalmente para criar aplicativos para Windows, jogos (com o Unity) e sistemas web. Ela é orientada a objetos, simples de aprender e poderosa para desenvolver uma variedade de aplicações.
  • Cloud Computing: É uma “folha de estilo” utilizada para definir o aspecto visual de páginas web que utilizam linguagens de marcação (ex: HTML).
  • CSS (Cascading Styles Sheet): Computação em nuvem, que permite o acesso remoto a servidores e armazenamento de dados pela internet.
  • Cluster: Arquitetura de sistema capaz combinar vários computadores para trabalharem em conjunto para que, em muitos aspectos, eles possam ser vistos como um único sistema.
  • Customer Experience (CX): Disciplina muito próxima do Design UX que cuida da experiência do consumidor em toda a sua jornada de compra.
  • Code Review: Ato de analisar o código de outra pessoa e fazer críticas construtivas a respeito de funcionalidades utilizadas e boas práticas de desenvolvimento.
  • Commitar: É o ato de salvar ou registrar as mudanças feitas no código-fonte em um sistema de controle de versão, como o Git. Isso cria um "marco" com uma descrição das alterações, permitindo rastrear o histórico do projeto.
  • Cookies: Pequenos arquivos enviados por um site para o navegador do usuário, que ficam armazenados no computador. Eles são feitos para guardar dados específicos de um cliente ou website, para serem acessados futuramente pelo servidor web de maneira mais rápida.
  • Copy/Copywriting: Advindo da Publicidade, diz respeito ao texto escrito com a finalidade de venda. Em Design UX, é comumente usado em relação a textos e diálogos em interfaces.

  • D
  • Dashboard: Painel que centraliza uma série de informações, como indicadores e métricas. É muito útil no meio empresarial, já que é projetado para ser atualizado em tempo real e de forma automática.
  • Dev: Abreviação de developer (do inglês). Pessoa desenvolvedora, programadora.
  • DevOps: É uma abordagem que integra as equipes de desenvolvimento (Dev) e operações (Ops) para melhorar a colaboração, automação e eficiência no ciclo de vida do software, desde o desenvolvimento até a produção. O objetivo é entregar software de forma mais rápida e confiável.
  • DNS (Domain Name System): Traduzido para a Língua Portuguesa, DSN significa Sistema de Nomes e Domínios. Se trata de um sistema responsável por localizar e traduzir para números IP os endereços dos sites que digitamos nos navegadores. Ele irá reconhecer seus números de IP para fazer uma conexão.

  • E

  • F
  • Frontend: A parte do desenvolvimento de software que envolve a criação da interface do usuário de um site ou aplicativo.

  • G
  • Git: Um sistema de controle de versão distribuído usado para gerenciar o código-fonte de projetos de software.

  • H
  • Hard skills: Habilidades técnicas, que podem ser mensuradas de forma mais objetiva (diferentemente das Soft Skills).
  • HTML: HTML (Hypertext Markup Language), é uma linguagem de marcação utilizada nas páginas web.

  • I
  • Iteratividade: É a capacidade de um usuário interagir e influenciar o conteúdo ou a experiência de um sistema, como em aplicativos, sites ou jogos, onde as ações do usuário geram respostas ou mudanças no ambiente.

  • J
  • Java: É uma linguagem de programação popular e versátil, usada para criar aplicativos para diferentes plataformas, como web, dispositivos móveis e sistemas empresariais. Ela é conhecida por ser fácil de aprender, segura e por permitir que o código seja executado em qualquer dispositivo com a Java Virtual Machine (JVM).
  • JavaScript: É uma linguagem de programação usada principalmente para criar interatividade em páginas web. Com ela, é possível manipular conteúdo, responder a eventos do usuário e atualizar a página sem recarregá-la. É essencial para o desenvolvimento de sites dinâmicos e aplicativos web.

  • K
  • Kanban: é um método visual de gerenciamento de tarefas, onde as atividades são organizadas em colunas (como "A Fazer", "Em Progresso" e "Concluído") para acompanhar o fluxo de trabalho. O objetivo é melhorar a eficiência, controlar o progresso e garantir que o trabalho flua de maneira contínua e sem sobrecarregar a equipe.

  • L
  • LAN (Local Area Network): É uma rede local que tem por finalidade a troca de dados dentro um mesmo espaço físico. Se trata de uma conexão de dispositivos dentro de uma área específica. O limitador da rede LAN é uma faixa de IP restrita à mesma, com uma máscara de rede comum.
  • Landing Page: Página de destino (ou “aterrissagem”) criada com o principal intuito de conversão do usuário: compra, cadastro, resposta, etc.
  • Lib: Abreviação de library, que significa biblioteca na Língua Inglesa. É um conjunto de funcionalidades ou programas que podem ser utilizados no desenvolvimento de projetos maiores.
  • Linguagem de alto nível: É a linguagem de programação que tem um nível de abstração elevado, se aproximando à linguagem humana.
  • Linguagem de baixo nível: É a linguagem de programação com sequências de instruções binárias entendidas pelo processador do computador.
  • LGPD: A Lei Geral de Proteção de Dados Pessoais (LGPD) é uma lei que define critérios obrigatórios para coleta, processamento e armazenamento de dados pessoais. Ela serve para proteger os dados do cidadão do uso indevido, comercialização e vazamento desses dados.
  • Loop: Repetição de um trecho de código, geralmente associado a um erro na execução do programa.

  • M
  • Metodologias Ágeis: Metodologias usadas para gestão do projeto, atividades, prazos, responsáveis e dimensionamento das entregas. Muito aplicadas em produtos digitais.
  • Mobile: Toda tecnologia que permite a utilização pelo usuário enquanto ele está em movimento. Não está restrita a smartphones.

  • N
  • Cloud Computing: Computação em nuvem, que permite o acesso remoto a servidores e armazenamento de dados pela internet.

  • O
  • Cloud Computing: Computação em nuvem, que permite o acesso remoto a servidores e armazenamento de dados pela internet.

  • P
  • Product Owner: É o responsável por definir e priorizar as funcionalidades de um produto, garantindo que a equipe de desenvolvimento trabalhe nas tarefas mais importantes para atender às necessidades dos usuários e alcançar os objetivos do negócio.
  • PHP: É uma linguagem de programação usada principalmente para desenvolver sites e aplicações web dinâmicas. Ela é executada no servidor e permite criar páginas que interagem com bancos de dados, formulários e outros recursos online.
  • POO (Programação Orientada a Objetos): É um paradigma de Programação que tenta aproximar a vida real da programação, tornando mais intuitivo e fácil de entender, além de poder repartir ainda mais o código, dividindo-o cada vez melhor.
  • Polimorfismo: É um conceito da programação orientada a objetos que permite que objetos de diferentes classes sejam tratados de forma uniforme, ou seja, que um mesmo método ou função possa ter comportamentos diferentes dependendo do tipo de objeto que o invoca.
  • Pull request: É uma solicitação feita por um desenvolvedor para integrar suas mudanças no código a um repositório principal. Ele permite que outros revisem e aprovem as alterações antes de serem mescladas ao projeto.
  • Python: É uma linguagem de programação de alto nível, fácil de aprender e muito usada em áreas como desenvolvimento web, análise de dados, inteligência artificial e automação. É conhecida por sua sintaxe simples e poderosa, permitindo escrever código de forma rápida e eficiente.

  • Q
  • Query: Processo de extração de dados de um banco de dados. Em outras palavras, se trata de um pedido de uma informação ou de um dado para um banco de dados e sua apresentação em uma forma adequada ao uso.

  • R
  • Roadmap: Ferramenta visual consistente em um mapa que aponta como será o projeto em cada etapa da sua evolução. É uma espécie de bússola que alinha as ações dos stakeholders, visando o estágio final do produto.
  • Ruby:É uma linguagem de programação de alto nível, conhecida por sua sintaxe simples e elegante. É frequentemente usada no desenvolvimento web, especialmente com o framework Ruby on Rails, que facilita a criação de aplicações web rápidas e eficientes.
  • React:É uma biblioteca JavaScript usada para construir interfaces de usuário, especialmente em aplicações web. Ela permite criar componentes reutilizáveis e gerenciar o estado da aplicação de forma eficiente, tornando o desenvolvimento de interfaces interativas mais rápido e organizado.

  • S
  • Scrum: É um framework ágil de gerenciamento de projetos, geralmente usado em desenvolvimento de software. Ele organiza o trabalho em ciclos curtos chamados sprints (geralmente de 2 a 4 semanas), onde a equipe foca em entregar um conjunto de funcionalidades. Scrum envolve papéis definidos (como Product Owner, Scrum Master e Time de Desenvolvimento) e cerimônias regulares (como reuniões diárias, revisão e retrospectiva) para garantir a colaboração e a melhoria contínua.
  • Sprint: É um ciclo curto de trabalho, geralmente de 1 a 4 semanas, onde uma equipe foca em concluir um conjunto específico de tarefas ou funcionalidades dentro de um projeto. É usado em metodologias ágeis como o Scrum.
  • Scrum Master: É o profissional responsável por garantir que a equipe siga as práticas do Scrum, removendo obstáculos e facilitando as reuniões e o trabalho em equipe, para que o processo seja eficiente e o projeto avance de forma produtiva.
  • Software: Coleção de dados ou instruções que informam a um sistema como ele deve trabalhar. Nada mais é do que um programa que você acessa no celular, tablet, computador ou qualquer outro dispositivo eletrônico. Pode ir desde sistemas operacionais, como Windows, macOS, iOS e Android aos apps que você usa todos os dias.
  • Soft Skills: Habilidades comportamentais de um indivíduo, intimamente ligadas à equipe e ao ambiente de trabalho. Exemplos: comunicação, trabalho em equipe, oratória etc.
  • Stack: É um conjunto de tecnologias ou ferramentas usadas juntas para desenvolver um produto ou solução. No contexto de desenvolvimento de software, um "tech stack" pode incluir linguagens de programação, frameworks, bancos de dados e outras ferramentas necessárias para criar uma aplicação.
  • Stakeholders: Qualquer parte interessada em um projeto, seja interna ou externa ao negócio: gestores, clientes, colaboradores, concorrentes, etc.
  • Skeumorfismo: Estilo de Design que se baseia nos elementos da vida real, utilizando metáforas destes elementos nas interfaces. Era o estilo utilizado pela Apple, mas foi modificado a partir do iOS7.
  • Sitemap: Trata-se do arquivo que elenca as páginas de um projeto de website, organizadas hierarquicamente de acordo com a ordem de navegação.

  • T
  • Cloud Computing: Computação em nuvem, que permite o acesso remoto a servidores e armazenamento de dados pela internet.

  • U
  • UI Design: É o processo de criar a interface visual de um produto, como sites ou aplicativos, focando na aparência e na experiência do usuário, incluindo botões, menus, cores e layouts. O objetivo é tornar a interação agradável e intuitiva.
  • UX Design: Área que cuida da User Experience (Experiência do Usuário). Trabalha para que a interação com o produto (ou seja, a sua experiência) seja fácil, marcante e eficiente, entre diversos outros aspectos.
  • UX/UI Design: Profissional responsável pela experiência e navegação dos usuários em multiplataformas como websites e aplicações mobile.
  • URL (Uniform Resource Locator): Traduzida para a Língua Portuguesa, significa Localizador Padrão de Recursos. Refere-se ao endereço de um recurso na rede como a internet. Entenda a URL como o endereço que você compartilha com os outros para que cheguem até o seu site ou blog.
  • User Flow: Traduzindo, fluxo do usuário. Diagrama que mapeia todo o fluxo de telas do site ou aplicativo, com o objetivo de identificar os possíveis “caminhos” a serem percorridos pelo usuário.

  • V
  • Versionar: Criar uma nova versão do software. O processo de versionamento ajuda a documentar inclusões, alterações e exclusões de funcionalidades e registra quando cada função foi ao ar, sendo possível resgatar versões anteriores em caso de erros no processo de publicação.
  • VPN (Virtual Private Network): Traduzida para a Língua Portuguesa, Rede Privada Virtual. VPN é uma rede de comunicação privada construída sobre a infraestrutura de uma rede pública. Essa é uma forma de conectar dois computadores através de uma rede pública, como a Internet. Mas a VPN fornece autenticação e confidencialidade na transmissão de dados, além de protocolos criptografados, o que as torna mais seguras e confiáveis.

  • W
  • Web: Sistema de informações ligadas através de texto, vídeo, som e outras animações digitais que permitem ao usuário acessar conteúdos através da internet.
  • Wireframe: É um “rabisco”, um esboço simples de uma interface com baixa fidelidade. Tem o intuito de estruturar e validar ideias.
  • WAU: Weekly Active Users (usuários ativos semanalmente). Métrica que verifica a frequência semanal de envolvimento de usuários em sites, softwares e aplicativos.

  • X
  • XP: Sigla que significa, em português, Limite de Trabalho em Movimento. Determina a quantidade de trabalho máxima que pode existir em cada etapa do fluxo de trabalho, com o propósito de identificar a ineficiência e aumentar a produtividade.

  • Y
  • Cloud Computing: Computação em nuvem, que permite o acesso remoto a servidores e armazenamento de dados pela internet.

  • Z
  • Cloud Computing: Computação em nuvem, que permite o acesso remoto a servidores e armazenamento de dados pela internet.
